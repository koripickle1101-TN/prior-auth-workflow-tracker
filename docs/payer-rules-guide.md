# Payer rules guide

This guide uses simulated payer rules for portfolio demonstration.

No real patient information is included.

## BCBS TN

Operational focus:

Authorization status monitoring, portal follow-up, and timely documentation review.

Suggested follow-up rule:

Check payer portal by day 3. Escalate by day 5 when the request is complete and still pending.

Common defect sources:

- missing medical necessity language
- service code mismatch
- incomplete provider note
- delayed payer response

## TennCare MCO

Operational focus:

Documentation completeness, referral coordination, and patient access risk.

Suggested follow-up rule:

Confirm required documentation before submission. Track high-risk services daily.

Common defect sources:

- missing clinical note
- incomplete referral details
- wrong service category
- unclear next action owner

## Medicare Advantage

Operational focus:

Service-specific documentation, medical necessity review, and payer portal monitoring.

Suggested follow-up rule:

Review documentation requirements before submission. Monitor aging requests every 48 hours.

Common defect sources:

- missing medical necessity support
- incomplete order detail
- payer-specific rule mismatch
- delayed status update

## Commercial payer

Operational focus:

Form accuracy, payer-specific process steps, and proof of submission.

Suggested follow-up rule:

Confirm receipt within 48 hours. Escalate when payer status is unclear after 5 days.

Common defect sources:

- wrong form
- missing clinical attachment
- unclear payer routing
- missing confirmation number
