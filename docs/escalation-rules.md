# Escalation rules

## Tier 1

Criteria:

- high patient access risk
- service scheduled within 72 hours
- authorization age above 5 days
- missing documentation tied to a scheduled service
- payer response delayed after follow-up

Required action:

Assign owner, document next action, and follow up same day.

## Tier 2

Criteria:

- medium patient access risk
- authorization age between 3 and 5 days
- submitted request pending payer review
- documentation appears complete
- no immediate scheduling impact

Required action:

Monitor status every 48 hours and update next action owner.

## Tier 3

Criteria:

- low patient access risk
- approved request
- routine follow-up
- no documentation gap
- no escalation needed

Required action:

Confirm scheduling handoff and close the authorization loop.

## Escalation ownership

Every authorization should have:

- current owner
- next action
- payer follow-up date
- documentation readiness status
- patient access risk level
- escalation status

## Operating rule

A request should not sit in the queue without ownership.

If no one owns the next action, the workflow is already unstable.
