# Prior Authorization Workflow Tracker

Simulated healthcare operations tracker for prior authorization workflows, documentation readiness, payer follow-up, escalation ownership, and patient access risk.

This project was created by Kori Pickle as a healthcare operations portfolio asset. It uses simulated data only. No PHI is included.

## Project purpose

Prior authorization problems often begin before a denial appears. This tracker organizes the operational work that determines whether an authorization moves cleanly or stalls.

The project tracks:

- open authorization requests
- average authorization age
- requests over 5 days
- missing documentation
- escalation ownership
- payer follow-up backlog
- patient access risk
- authorization denial risk

## Workflow questions answered

1. Which authorizations are aging?
2. Which payer is creating the most delay?
3. Which requests are missing documentation?
4. Who owns the next action?
5. Which cases risk delaying care?
6. Which requests need escalation?
7. Which upstream defect is causing the delay?
